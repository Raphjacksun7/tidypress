DocsMint turns a `docs/` folder into a small static site for documentation and writing.

You write markdown. DocsMint prepares an internal Astro workdir, renders the site, runs Pagefind during production builds, and leaves you with static files.

## Install

<Tabs labels={["npm", "pnpm", "python"]}>
<Tab>

```bash
npm install -g docsmint
```

</Tab>
<Tab>

```bash
pnpm add -g docsmint
```

</Tab>
<Tab>

```bash
pip install docsmint
```

</Tab>
</Tabs>

<Callout>
Node.js 22.12 or newer is required for rendering. The Python package can run Python-native helpers, but site builds still use the Node.js CLI and Astro engine.
</Callout>

## Start a project

From the root of your project:

```bash
docsmint init
docsmint dev
```

Open `http://localhost:4321`.

`docsmint init` creates a `docs/` folder:

```txt
docs/
├── docsmint.config.ts
├── public/
│   └── images/
└── src/
    └── content/
        ├── docs/
        │   └── getting-started.md
        ├── writing/
        │   └── hello.md
        └── pages/
```

The default starter seeds one docs page and one writing post so you can see the file shape before replacing them.

Use the custom starter when you also want an example custom collection:

```bash
docsmint init --preset custom
```

## Write content

Put documentation pages in `docs/src/content/docs/`:

```md
---
title: Install
description: Install and configure the project.
order: 1
---

## Requirements

Write the page in markdown.
```

Put dated posts in `docs/src/content/writing/`:

```md
---
title: Release notes
date: "2026-05-22"
description: Notes from the latest release.
---

What changed and why.
```

## Build and preview

```bash
docsmint build
docsmint preview
```

Build output is written to:

```txt
docs/.docsmint/dist/
```

Search is generated during `docsmint build`, so the search modal is available in `preview`, not in the dev server.

## What `.docsmint/` is

`.docsmint/` is generated. Do not edit it by hand.

During dev, DocsMint links your content and config into the workdir so Astro can reload changes. During build, DocsMint copies the content into the workdir and writes static output.

```txt
your project
└── docs/
    ├── docsmint.config.ts      # your config
    ├── src/content/            # your markdown
    └── .docsmint/              # generated workdir and dist output
```

If a local build behaves strangely after an engine update, run:

```bash
docsmint clean
docsmint build
```

## Static assets

Put files that should be served as-is in `docs/public/`:

```txt
docs/public/
├── images/diagram.png          # /images/diagram.png
├── downloads/spec.pdf          # /downloads/spec.pdf
├── robots.txt                  # /robots.txt
└── favicon.svg                 # /favicon.svg
```

Use absolute paths in markdown:

```md
![Architecture diagram](/images/diagram.png)
[Download the spec](/downloads/spec.pdf)
```

`downloads/` is only an example folder. Any file under `public/` is served from the site root.
