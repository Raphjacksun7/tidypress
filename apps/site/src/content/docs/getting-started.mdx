</Callout>

## Start a project

From the root of your project:

```bash
npx docsmint init
npx docsmint dev
```

Open `http://localhost:4321`.

`init` creates a `docs/` folder:

```txt
docs/
├── docsmint.config.ts
├── public/
│   └── images/
└── src/
    └── content/
        ├── docs/
        │   └── getting-started.md
        ├── writing/
        │   └── hello.md
        └── pages/
```

`init` defaults to the **lab** preset: writing and projects on the home page, docs disabled. Replace the seeded markdown with your own content.

Other presets:

| Preset | Shape |
|--------|--------|
| `lab` (default) | writing + projects |
| `persona` | hero bar, projects, writing, about page |
| `blog` | writing only (docs and pages off) |
| `docs-writing` | docs + writing |
| `custom` | docs + writing + a `playbooks` content collection |

`default` is an alias for `lab`.

```bash
npx docsmint init --preset blog
npx docsmint init --preset persona
npx docsmint init --preset docs-writing
npx docsmint init --preset custom
```

## Write one docs page

Put documentation pages in `docs/src/content/docs/`:

```md
---
title: Install
description: Install and configure the project.
order: 1
---

## Requirements

Write the page in markdown.
```

Put dated posts in `docs/src/content/writing/`:

```md
---
title: Release notes
date: "2026-05-22"
description: Notes from the latest release.
---

What changed and why.
```

## Build and preview

```bash
npx docsmint build
npx docsmint preview
```

Build output is written to:

```txt
docs/build/
```

Search is generated during `docsmint build`, so the search modal is available in `preview`, not in the dev server.

Deploy that directory with your static host, CI job, or `docsmint deploy`.

## What `.docsmint/` is

`.docsmint/` is generated. Do not edit it by hand.

During dev, DocsMint links your content and config into the workdir so Astro can reload changes. During build, DocsMint copies the content into the workdir and writes static output.

```txt
your project
└── docs/
    ├── docsmint.config.ts      # your config
    ├── src/content/            # your markdown
    └── .docsmint/              # generated workdir and dist output
```

If a local build behaves strangely after an engine update, run:

```bash
npx docsmint clean
npx docsmint build
```

## Static assets

Put files that should be served as-is in `docs/public/`:

```txt
docs/public/
├── images/diagram.png          # /images/diagram.png
├── downloads/spec.pdf          # /downloads/spec.pdf
├── robots.txt                  # /robots.txt
└── favicon.svg                 # /favicon.svg
```

Use absolute paths in markdown:

```md
![Architecture diagram](/images/diagram.png)
[Download the spec](/downloads/spec.pdf)
```

`downloads/` is only an example folder. Any file under `public/` is served from the site root.
