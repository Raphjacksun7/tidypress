Locale-prefixed routes and UI labels.

## Configure locales

```ts
import { defineConfig } from 'tidypress/config'

export default defineConfig({
  name: 'my-project',
  i18n: {
    defaultLocale: 'en',
    locales: ['en', 'fr'],
  },
})
```

Routes:

```txt
/docs/getting-started
/fr/docs/getting-started
/writing
/fr/writing
```

The default locale stays at the root path. Non-default locales use a locale prefix.

## Content

```txt
docs/src/content/
├── docs/
│   ├── getting-started.md
│   └── fr/getting-started.md
└── writing/
    ├── hello.md
    └── fr/hello.md
```

If a non-default locale is missing a page, TidyPress falls back to the default-locale content for that route.

## UI strings

```ts
i18n: {
  defaultLocale: 'en',
  locales: ['en', 'fr'],
  strings: {
    fr: {
      searchLabel: 'rechercher',
      searchPlaceholder: 'Rechercher...',
      docsTitle: 'Docs',
      writingTitle: 'Articles',
      onThisPageLabel: 'Sur cette page',
    },
  },
}
```

Keys:

- `searchLabel`
- `searchPlaceholder`
- `searchEmptyLabel`
- `searchNoResultsLabel`
- `docsTitle`
- `writingTitle`
- `allDocsLabel`
- `allWritingLabel`
- `onThisPageLabel`
- `editThisPageLabel`
- `languageLabel` (footer language switcher when `i18n.locales` has more than one locale)

The switcher renders in the footer main band (`main.end`). See [Configuration → Footer](./configuration#footer).

## Search

[Pagefind](https://pagefind.app/) indexes the built pages. The search UI scopes localized results to the active path when possible.
